Monitorización con Linux y Docker

Implementación de un sistema integral de monitorización ("Ulls que tot ho veuen") basado en software libre. El proyecto utiliza Docker para contenerizar servicios y orquestar una solución escalable sobre Ubuntu Server 22.04 LTS.

El núcleo del sistema es Zabbix para la recolección de métricas y alertas en tiempo real, integrado con Grafana para la visualización de datos mediante dashboards interactivos. El objetivo principal fue asegurar la disponibilidad de los servicios y permitir una respuesta proactiva ante incidentes.

Detalles del proyecto

Se desplegaron agentes Zabbix en los clientes para monitorizar CPU, RAM y red. Se realizaron pruebas de estrés ("Stress-ng") simulando cargas del 100% de CPU, verificando que el sistema detectaba la anomalía en menos de 10 segundos, enviaba alertas por correo y marcaba el problema como resuelto automáticamente al finalizar la carga.

Resultados alcanzados

  • Detección de "High CPU Load" en menos de 10 segundos
  • Visualización gráfica unificada en Grafana
  • Alertas automatizadas vía correo electrónico
  • Despliegue modular y portable gracias a Docker Compose
